# Museum Conservation Tracker

## Summary
Museum Conservation Tracker is a full-stack workspace for managing museum conservation workflows with live dashboards, seeded operational records, and room to extend into production features.

## Problem
Museum Conservation Tracker addresses a real-world gap in conservation tracking, where teams still rely on spreadsheets, email, or delayed follow-up.

## Frontend Features
- Museum Conservation Tracker artifact dashboard
- Museum Conservation Tracker treatment timeline
- Museum Conservation Tracker environment alerts

## Backend Features
- Museum Conservation Tracker artifact APIs
- Museum Conservation Tracker condition endpoints
- Museum Conservation Tracker event logs

## Tech Stack
- HTML
- CSS
- JavaScript
- Node.js
- Express
- PostgreSQL-ready schema

## Difficulty
Medium-Hard

## Run Locally
1. In `backend`, run `npm install` and then `npm run dev`.
2. In `frontend`, run `npm install` and then `npm run dev`.
3. Open the frontend URL shown by `serve`.

## API Endpoints
- `GET /api/health`
- `GET /api/overview`
- `GET /api/work-items`
- `GET /api/metrics`
- `POST /api/work-items`
